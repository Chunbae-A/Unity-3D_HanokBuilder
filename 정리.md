# HanokBuilder — Unity 런타임 UI 개발 정리

> 한국 전통공간 모듈러 3D 빌더 플러그인  
> Unity 6 (6000.3.16f1) / Input System 패키지 환경

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | HanokBuilder |
| 목표 | 한옥·전통공간 3D 에셋을 조립·편집하는 Unity 런타임 UI 툴 |
| 엔진 | Unity 6 (6000.3.16f1) |
| 입력 시스템 | Unity Input System 패키지 |
| 폰트 | NotoSansKR-Regular SDF (TextMeshPro) |
| 에셋 로드 방식 | `Resources.LoadAll` — `Assets/HanokBuilder/Resources/HanokAssets/` |


## 구현된 기능

### 1. 런타임 UI (HanokUIManager.cs)

- **Canvas 자동 생성** — `Start()`에서 코드로 전체 UI를 생성 (Inspector 수동 배치 없음)
- **CanvasScaler** — `ScaleWithScreenSize` 1280×720 기준 반응형
- **왼쪽 패널**: 한옥 에셋 목록 (스크롤 가능)
- **오른쪽 패널**: Transform 편집 (위치/회전/크기)
- **에셋 자동 로드**: `Resources/HanokAssets` 폴더 Prefab 자동 인식

### 2. 에셋 배치 (Spawn)

- 카메라 정중앙 레이캐스트로 지면 교점에 배치
- 교점 없으면 카메라 전방 10m 지점 Y=0 에 배치
- BoxCollider 자동 추가 (없는 경우) — Renderer bounds 기준 자동 크기 계산

### 3. 오브젝트 선택 (SelectableAsset.cs)

- `Mouse.current.leftButton.wasPressedThisFrame` (Input System 대응)
- UI 영역 클릭 시 무시 (`EventSystem.IsPointerOverGameObject`)
- FBX 자식 구조 대응 — 루트 및 모든 자식 Collider에 `SelectableAsset` 자동 부착
- `Root` 프로퍼티로 항상 루트 오브젝트 선택 보장

### 4. Transform 편집

| 기능 | 동작 |
|------|------|
| Position X/Y/Z | InputField 직접 입력 → `onEndEdit` 적용 |
| Rotation X/Y/Z | InputField 직접 입력 → `onEndEdit` 적용 |
| 빠른 회전 | −90° / +90° / 초기화 버튼 |
| Scale 균등 | InputField 입력 + 0.5× / 1× / 2× 프리셋 버튼 |
| 복제 | 선택 오브젝트 +X 2m 위치에 복제 |
| 삭제 | 선택 오브젝트 Destroy |
| 선택 해제 | selectedObject = null |

## 해결한 주요 이슈

### Issue 1 — `MidlineCenter` 컴파일 오류
- **원인**: Unity 6 TMP 버전에서 `TextAlignmentOptions.MidlineCenter` 제거됨
- **해결**: `TextAlignmentOptions.Center` / `Left` / `Right` 로 교체

### Issue 2 — Image + TextMeshProUGUI 동일 오브젝트 충돌
- **원인**: 같은 GameObject에 `Image`와 `TextMeshProUGUI` 동시 부착 불가 (둘 다 `Graphic` 상속)
- **해결**: 텍스트를 반드시 자식 GameObject에 분리

### Issue 3 — Input System 충돌
- **원인**: `UnityEngine.Input` 구형 API 사용 → Input System 패키지와 충돌
- **해결**: `Mouse.current.leftButton.wasPressedThisFrame` (New Input System API) 로 교체

### Issue 4 — 에셋 목록 미표시
- **원인 A**: `Resources.LoadAll<GameObject>` 미인식 → `Resources.LoadAll` + 타입 캐스팅으로 변경
- **원인 B**: FBX 파일을 Prefab으로 변환하지 않음 → Prefab으로 변환 필요
- **원인 C**: `ContentSizeFitter` + `VerticalLayoutGroup` 첫 프레임 레이아웃 미계산 → `ForceRebuildLayoutImmediate` 코루틴으로 강제 갱신

### Issue 5 — 에셋 배치 후 화면에 안 보임
- **원인**: 카메라 Position Z가 −6057 수준으로 이동되어 있음
- **해결**: Camera Position `(0, 5, -10)`, Rotation `(20, 0, 0)` 으로 재설정

### Issue 6 — 오브젝트 클릭 선택 안됨
- **원인**: FBX 계층구조에서 Collider가 자식 오브젝트에만 존재 → 루트에 SelectableAsset 없음
- **해결**: `AttachSelectable()` 함수로 루트 + 모든 자식 Collider에 SelectableAsset 일괄 부착

### Issue 7 — Awake vs Start 타이밍
- **원인**: `Awake()`에서 UI 빌드 시 일부 Unity 시스템 미초기화
- **해결**: `Start()`로 이동

## 현재 파일별 역할

### HanokUIManager.cs

```
담당: UI 전체 생성 및 로직
- BuildUI()         : Canvas, 패널, 스크롤뷰, 버튼 코드로 자동 생성
- FillEditPanel()   : 오른쪽 Transform 편집 패널 내용 생성
- LoadAssets()      : Resources/HanokAssets에서 Prefab 자동 로드 및 버튼 생성
- Spawn()           : 에셋 씬 배치 (레이캐스트 기반 위치 결정)
- SelectObject()    : 오브젝트 선택 및 편집 패널 동기화
- HandleClick()     : Input System 기반 씬 클릭 감지
- SyncInputs()      : 매 프레임 선택 오브젝트 Transform → InputField 동기화
```

### SelectableAsset.cs

```
담당: 씬 오브젝트 선택 가능 표시
- Init(manager, root) : UIManager 참조 및 루트 오브젝트 저장
- Root 프로퍼티       : 항상 루트 오브젝트 반환 (자식에 붙어도 루트 선택)
```

## 에셋 추가 방법

1. FBX 파일을 Hierarchy로 드래그 → 씬에 올라감
2. Hierarchy 오브젝트 우클릭 → `Prefab → Create Original Prefab`
3. 생성된 `.prefab` 파일을 `Assets/HanokBuilder/Resources/HanokAssets/` 로 이동
4. Play 시 왼쪽 목록에 자동 표시됨
